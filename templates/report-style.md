# Report Style Guide

Portable markdown + embedded CSS for **structured reports** generated by scripts or `writer-agent`. Uses HTML inside markdown for rich preview in Cursor / VS Code / GitHub.

**Golden reference:** `license_app/docs/cryptlex-breaking-change-manual-test-evidence-2026-07-20.md`

**Style resolution (in order):**

1. Target repo local convention (`docs/REPORT-STYLE.md` or existing styled report in repo)
2. This guide (`cursor-agents/templates/report-style.md`)
3. Prose tone from [`documentation-style.md`](./documentation-style.md) — layout uses embedded CSS here, not GitHub admonitions

---

## 1. When to use

| Report type | Examples | Audience |
|-------------|----------|----------|
| **Test report** | API/integration test runs, manual test evidence | QA, tech leads |
| **Impact report** | Breaking-change, migration, dependency audit | Architects, backend devs |
| **Audit report** | Codebase scan + runtime checks | Security, compliance, release |

Use this style when the deliverable has:

- Executive summary with counts or verdicts
- A wide summary table (many columns)
- Repeated item sections (test cases, findings, endpoints)
- TOC with anchor links

For module overviews, specs, ADRs, setup guides → use [`documentation-style.md`](./documentation-style.md).

---

## 2. Report skeleton

```markdown
# <Title>

<style>
/* Paste full CSS from §3 */
</style>

<p class="report-meta">
  <strong>Generated:</strong> <ISO-8601> &nbsp;|&nbsp;
  <strong>Environment:</strong> <env> (<code><base-url></code>)
</p>

## Table of Contents
- [Executive summary](#executive-summary)
- [Impact summary](#impact-summary)
- [Section A — …](#section-a--…)
- [Appendix — Index](#appendix--index)

## Executive summary
<!-- stats-wrap + conclusion-panel -->

## Impact summary
<!-- impact-table -->

<div class="section-header"><h2>Section A — …</h2></div>

<div class="report-item">
<h2 class="report-item-title"><span class="item-badge">R1</span> Item title</h2>
<!-- detail content -->
</div>
```

> **Class aliases:** Reports may use `.test-case` / `.tc-badge` (legacy) or `.report-item` / `.item-badge` (generic). CSS below includes both; prefer `.report-item` for new reports.

## 4. HTML patterns

### Metadata

```html
<p class="report-meta">
  <strong>Generated:</strong> 2026-07-20T08:06:36.998Z &nbsp;|&nbsp;
  <strong>Environment:</strong> local (<code>http://localhost:3000</code>)
</p>
```

### Context box

```html
<div class="change-box">
<h3>Scope / change</h3>
Before <span class="deadline">2026-09-01</span>, …
</div>
```

### Conclusion panel

```html
<div class="conclusion-panel">
<div class="conclusion-header">Overall conclusion</div>
<div class="conclusion-body">
<p class="conclusion-lead"><strong>Main finding</strong> in one sentence.</p>
<ul class="conclusion-list">
<li>Supporting point.</li>
</ul>
</div>
</div>
```

### Summary table + badges

```html
<table class="impact-table">
<colgroup><col style="width:12%"></colgroup>
<thead><tr><th>Item</th><th>Verdict</th></tr></thead>
<tbody>
<tr>
  <td class="col-endpoint"><code>GET /api/:id</code></td>
  <td class="col-verdict"><span class="verdict pass">PASS</span></td>
  <td class="col-code"><span class="code-update no">No</span></td>
  <td class="col-action"><span class="action none">None</span></td>
</tr>
</tbody>
</table>
```

### Section + item card

```html
<div class="section-header"><h2>Section A — …</h2></div>

<div class="report-item">
<h2 class="report-item-title"><span class="item-badge">R1</span> Title</h2>
<!-- tables, evidence -->
</div>
```

Legacy test reports may keep `.test-case` / `.tc-badge` / `C1-01` IDs — CSS supports both.

---

## 5. Content rules

- Redact secrets (tokens, keys) → type placeholders
- Sample long arrays (1–3 items)
- Normalize IDs as `:id` in summary tables
- TOC links for every section and item
- Do not invent verdicts or findings
- English unless project uses bilingual docs

---

## 6. Script / agent usage

```text
@writer-agent
Generate report using cursor-agents templates/report-style.md.
Output: docs/<topic>-report-<YYYY-MM-DD>.md
```

Report generators: emit §3 CSS at top; wrap each item in `.report-item`; use verdict badge classes.
